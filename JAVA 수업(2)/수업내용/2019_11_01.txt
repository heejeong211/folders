-참조변수
주소값을 가지고 있는 것이 참조변수 이다.

-참조변수에 null 대입
참조변수에 null를 넣게 되면 이어져왔던 클래스와의 관계를 끊어버리는 것이다.
좀 더 깊게 들어가서 설명하면,
가베지 컬렉터한테 명시적으로 난 필요없으니 이거 너가 회수해라 라고 알려주는 것.

-String클래스
스트링은 클래스이다. 클래스에는 데이터와 함수가 있다.
String str1 = "Happy";
String str1은 선언 한것이다. 그렇다고 String은 데이터타입이 아니다. 클래스이다.
str1은 참조형 변수이다. 그래서 안에 주소가 들어가야 한다. 
근데?? 객체생성을 하고 있지 않다. 그런데 주소가 들어간다.

-문자열
"Happy" 도 메모리주소에 올려야 한다. 
char은 문자 하나 2바이트
문자열을 2바이트씩 잡아주고 제일 마지막에 null이 들어간다.
H a p p y 에 대한 유니코드값이 들어감.
문자열이 시작되는 위치와 문자열이 끝나는 위치를 알아야 한다.
근데 문자열의 끝이 어딘지 모르니까 컴파일러가 알아서 null을 붙인다.

-문자열을 프린트하는 하는 방법
문자열의 첫번째 위치부터 문자열이 끝나는 null까지 프린트 한다.

-초기화 메소드
초기화란? 변수 선언을 하고 그 변수에 첫번째 값을 넣는 것.

-생성자
new 옆에 붙는 함수를 호출해야하는 함수는 클래스명과 같은 함수, 데이터타입이 없는 함수를 생성자라고 한다.
BankAccount() -> 함수인데 함수를 호출할 수 있음.
함수생성을 안했는데 함수를 호출할 수 있는 이유는 컴파일러가 알아서 만들어 주기 때문임.
클래스 이름과 함수 이름이 같음.
생성자 하나라도 있으면 디폴트 생성자 안만듦.

-생성자의 목적
변수 초기화가 목적임.

-컴파일은 클래스 단위 만큼 생김

-디폴트 생성자
생성자가 한개도 없을 때 
데이터타입도 없고 클래스명과는 같은데 인자도 없는 함수

- 생성자를 한개라도 있으면 컴파일러가 디폴트로 안만들어 줌.
A a = new A(6, 7); -> 얜 위에서 만들어 줌.
A b = new A(); -> 얘 안만들어 주면 에러남.

-클래스 이름의 첫 문자는 대문자로 시작한다.
카멜표기법(헝가리안 표기법)

-메소드와 변수의 이름의 첫 문자는 소문자로 시작한다.

-상수의 이름은 모든 문자를 대문자로 구성한다. 
둘 이상의 단어가 묶여서 하나의 이름을 이룰 때 단어 사이를 언더바로 연결한다.
